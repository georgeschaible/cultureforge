OVERALL SUMMARY:
  Total accessions: ~95
  VERIFIED (correct protein): ~30
  WRONG-PROTEIN (completely unrelated): ~45
  WRONG-MARKER (related but wrong gene): ~5
  Marker files entirely wrong (no correct sequences): aprAB, hao, soxB, pscA_fmoA, cyc2
  Marker files mostly wrong: pufLM, autotrophy, mcrA, cooS_cdhA, mtrC_omcB, nosZ, mcrBG

CONCLUSION: Approximately 50% of all marker reference accessions return wrong proteins.
This is because the accessions in fetch_markers.sh were generated from expected gene names
and organism names but never verified against UniProt's actual content. UniProt accession
numbers are stable identifiers but don't always correspond to the protein one expects from
the gene name in a given organism's genome annotation.

Despite this, detection has been working reasonably well because:
1. The few CORRECT sequences are strong enough to produce genuine BLAST hits
2. Wrong sequences mostly produce no hits or below-threshold noise
3. Some wrong sequences happen to be from the right organism and produce weak hits
   to related proteins that don't cross the positive_call threshold

However, this means detection sensitivity is severely reduced — many markers are
operating with 1-2 correct references instead of the intended 3-5.
