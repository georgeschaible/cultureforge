Provenance nuance (why match_method='manual', not 'species_name_exact'):
  The only BacDive strain record for this genome is 11901, whose primary
  species field reads "Picrophilus oshimae" — BacDive files the
  P. torridus type strain under P. oshimae (an LPSN synonym view).
  Strain identity is nonetheless certain: 11901 is DSM 9790 / KAW 2/3,
  and BacDive's own genome cross-reference inside record 11901 reads
  "Picrophilus torridus DSM 9790" (= GCF_000008265.1, gid 26). Recording
  this as 'species_name_exact' would be factually untrue (the species
  strings differ), so it is recorded as 'manual' at confidence 0.95. The
  upstream-mirrored bacdive_cache row for 11901 was deliberately left
  unchanged (a faithful BacDive mirror); the rationale lives in the audit
  erratum and this message because organism_to_bacdive has no notes
  column.
