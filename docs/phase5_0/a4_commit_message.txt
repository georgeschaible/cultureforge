A4: replace stale Scalindua species-name guard with evidence-based check

What was wrong:
  compose_recipe.py:2005-2020 ("E.1 — Scalindua MAG completeness"), invoked
  from _apply_limitations_flags, force-escalated ANY species whose name
  contained "scalindua", unconditionally, with a hardcoded rationale
  ("predicted proteome lacks hzsA/hdh"). That rationale was a defensive
  fossil from the pre-2026-05-05 gid-30 state (when gid 30 held
  Salmonella-contaminated data). After the audit-corrected MAG re-download,
  the guard became stale: gid 30 (Ca. Scalindua japonica husup-a2) and
  gid 1105 (Ca. Scalindua brodae) are correctly detected as anammox at
  capability 0.95 with strong positive_call hzsA (~64% id, bs~1070) and hdh
  (~77% id, bs~970) hits, _compose_anammox_recipe builds the correct
  anaerobic recipe — and the guard then discarded that correct recipe with
  a reason the marker evidence directly contradicts.

What changed:
  The species-name predicate is replaced with a marker-evidence predicate:
  escalate ONLY when the anammox capability/mode is asserted AND both hzsA
  and hdh lack a positive_call=1 row in genome_diagnostic_markers (the
  genomic signature of a genuinely incomplete anammox MAG). The defensive
  intent is preserved honestly — the escalation reason now states the
  evidence that actually triggered it instead of a false hardcoded claim.
  Marker presence uses the codebase's existing positive_call convention,
  mirroring the _compose_anme_recipe pattern (compose_recipe.py:471-482).
  conn is threaded into _apply_limitations_flags (its sole call site,
  line 222); no other limitation-flag rule is changed. The anammox-asserted
  gate is essential: _apply_limitations_flags runs for all genomes, so an
  ungated hzsA/hdh-absence check would falsely escalate every non-anammox
  organism.

  Also (documentation only): data/pathway_definitions.json anammox
  limitation_summary rewritten — the "Scalindua-type hzsA too divergent for
  BLAST or HMM detection" claim is contradicted by ~60-77% identity hits at
  HEAD and was corrected to describe the current evidence-based guard.
